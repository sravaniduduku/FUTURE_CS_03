 API Security Risk Analysis

## Project Overview:

This project was completed as part of the **Future Interns Cyber Security Internship – Task 3: API Security Risk Analysis**. The objective of this task is to analyze API endpoints for potential security risks, identify vulnerabilities, and provide recommendations to improve API security.

The project demonstrates the assessment of API security controls such as authentication, authorization, data exposure, HTTPS implementation, security headers, and information disclosure risks.

## Objectives:

* Analyze API endpoints for common security vulnerabilities.
* Evaluate authentication and authorization mechanisms.
* Identify sensitive data exposure risks.
* Check for secure communication using HTTPS.
* Review security-related HTTP headers.
* Generate a professional security assessment report.

## Features:

* HTTPS Security Verification
* HTTP Response Header Analysis
* Authentication Assessment
* Information Disclosure Detection
* HTTP Status Code Monitoring
* Automated Security Report Generation

## Security Checks Performed:

### HTTPS Validation:

Verifies whether the API uses secure HTTPS communication.

### Security Headers Analysis:

Checks for important security headers such as:

* Content-Security-Policy
* X-Frame-Options
* X-Content-Type-Options
* Strict-Transport-Security

### Authentication Assessment:

Determines whether the API endpoint requires authentication.

### Information Disclosure Detection:

Scans API responses for potentially sensitive information exposure.

### Status Code Analysis:

Monitors HTTP response codes and reports endpoint behavior.
